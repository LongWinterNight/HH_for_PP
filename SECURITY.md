## Известные ограничения безопасности

### Текущая версия (1.1.0)

| Компонент | Ограничение | Статус |
|-----------|-------------|--------|
| Веб-интерфейс | CORS разрешён для всех источников | ⚠️ Для продакшена требуется ограничение |
| Аутентификация | Отсутствует | ⚠️ Планируется в v1.2.0 |
| HTTPS | Не требуется (локальное использование) | ✅ Для локального использования ОК |
| База данных | SQLite без шифрования | ⚠️ Для продакшена рассмотреть PostgreSQL |

### Рекомендации для продакшена

1. **Ограничьте CORS**
   ```python
   app.add_middleware(
       CORSMiddleware,
       allow_origins=["https://yourdomain.com"],
       ...
   )
   ```

2. **Добавьте аутентификацию**
   - JWT токены
   - OAuth2

3. **Используйте HTTPS**
   - SSL сертификаты
   - Принудительное перенаправление

4. **Защитите базу данных**
   - PostgreSQL с ролями
   - Шифрование чувствительных данных

5. **Валидация входных данных**
   - Pydantic модели (уже используется)
   - Санитизация пользовательского ввода
