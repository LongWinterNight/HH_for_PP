# 📋 Инструкция по переносу и демонстрации проекта

## 🎯 Цель
Подготовить проект HH.ru Analytics для демонстрации на ноутбуке без интернета.

---

## 📦 Шаг 1: Подготовка на текущем компьютере

### 1.1 Проверка данных

```bash
cd f:\AI_projects\hh_analytics

# Проверить наличие данных в базе
python check_functionality.py

# Убедиться что в базе есть вакансии (минимум 50+)
```

### 1.2 Очистка проекта

```bash
# Удалить временные файлы
del /s /q __pycache__
del /s /q *.pyc
del /s /q .pytest_cache

# Очистить логи (опционально)
del logs\*.log

# Удалить старые отчёты (опционально)
del reports\*.xlsx
del reports\*.pdf
```

### 1.3 Проверка .env файла

**Важно:** `.env` файл НЕ коммитится в Git! Создайте `.env` на целевом компьютере.

```env
# HH.ru API токен (если нужен парсинг)
HH_API_TOKEN=your_token_here

# База данных
DB_PATH=data/hh_analytics.db

# Логирование
LOG_LEVEL=INFO
LOG_FILE=logs/hh_analytics.log
```


```
├── hh_analytics/
│   ├── src/                    # Исходный код
│   ├── web/                    # Веб-приложение
```

## 🖥️ Шаг 3: Настройка на целевом ноутбуке

### 3.1 Проверка Python

```bash
python --version
# Должно быть Python 3.10 или выше
```

Если Python не установлен:
1. Скачать с https://www.python.org/downloads/
2. Установить Python 3.10+
3. ✅ Поставить галочку "Add Python to PATH"

### 3.2 Создание виртуального окружения

**Windows:**
```bash
cd hh_analytics
python -m venv venv
venv\Scripts\activate
```

**Linux/macOS:**
```bash
cd hh_analytics
python3 -m venv venv
source venv/bin/activate
```

### 3.3 Установка зависимостей

```bash
pip install -r requirements.txt
```

**Время установки:** ~2-5 минут (в зависимости от интернета)

### 3.4 Создание .env файла

```bash
# Windows
copy .env.example .env

# Linux/macOS
cp .env.example .env
```

Открыть `.env` и при необходимости изменить пути.

## 🔧 Шаг 6: Решение проблем

### Проблема 1: Порт 8000 занят

**Симптомы:**
```
Error: [Errno 10048] error while attempting to bind on address ('0.0.0.0', 8000)
```

**Решение:**
```bash
# Windows - найти процесс
netstat -ano | findstr :8000

# Убить процесс (заменить PID)
taskkill /F /PID <PID>

# Или запустить на другом порту
python web/app/main.py --port 8001
```

### Проблема 2: ModuleNotFoundError

**Симптомы:**
```
ModuleNotFoundError: No module named 'fastapi'
```

**Решение:**
```bash
# Переустановить зависимости
pip uninstall -y -r requirements.txt
pip install -r requirements.txt
```

### Проблема 3: База данных пуста

**Симптомы:**
```
total_vacancies: 0
```

**Решение:**
```bash
# 1. Проверить путь к БД в .env
# 2. Скопировать базу с исходного компьютера
#    data/hh_analytics.db

# 3. Или запустить парсер
python optimized_parser.py
```

### Проблема 4: Браузер не открывается

**Симптомы:**
```
Connection refused / Не удаётся подключиться
```

**Решение:**
1. Проверить что сервер запущен (см. консоль)
2. Попробовать http://127.0.0.1:8000
3. Проверить брандмауэр Windows
4. Перезапустить сервер

## 🌐 Шаг 8: Публикация на GitHub

### 8.1 Инициализация репозитория

```bash
cd f:\AI_projects\hh_analytics

# Если Git ещё не инициализирован
git init

# Добавить все файлы
git add .

# Сделать коммит
git commit -m "Initial commit: HH.ru Analytics v1.1.0"

# Добавить удалённый репозиторий
git remote add origin https://github.com/LongWinterNight/HH_for_PP.git

# Запушить
git push -u origin main
```

### 8.2 Обновление существующего репозитория

```bash
# Проверить статус
git status

# Добавить изменения
git add .

# Закоммитить
git commit -m "Add filters and export functionality v1.1.0"

# Запушить
git push origin main
```

### 8.3 Что должно быть в репозитории

```
✅ Исходный код (src/, web/)
✅ Конфигурация (config.yaml, requirements.txt)
✅ Документация (README.md)
✅ База данных (data/hh_analytics.db) — опционально
✅ .env.example
✅ .gitignore

❌ .env (с секретами)
❌ venv/
❌ __pycache__/
❌ logs/*.log
❌ reports/*.xlsx (большие файлы)
```

### 8.4 GitHub Pages (опционально)

Для демонстрации можно использовать GitHub Pages:

1. Settings → Pages
2. Source: Deploy from branch
3. Branch: main, folder: / (root)
4. Save

**Но:** FastAPI требует сервер, так что GitHub Pages не подойдёт для backend. Используйте для статической документации.
